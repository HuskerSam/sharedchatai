The chat request page allows users to communicate with the AI and receive responses for their queries. The tickets provide useful information such as the message owner and time since the message was sent, while the select box and rerun button give users more control over the AI's response. The token visualizer helps users understand how the model operates and provides insight into how text is broken down into its constituent parts. The engine section allows users to choose the appropriate version of the Chat GPT API for the task at hand, with the completion token and temperature fields available for adjusting the output.


  • Messages from user and AI responses
  • Tickets that show the message owner, time since message was sent, and a delete button for the message owner
  • Select box for excluding messages from being sent to the AI
  • Rerun button for resubmitting included messages
  • Engine section for selecting appropriate version of the Chat GPT API for the task at hand
  • Chat GPT API includes completion token (max_tokens) and temperature fields to adjust the output
  • Token visualizer at the top of the text area for understanding model's tokenization process
  • Three token counts on the page:
    • Total token count for all messages sent so far
    • Token count for the response sent back for individual tickets
    • Estimated total token count for new messages
