Intro to Promptplus AI

Overview

PromptPlus Ai is a collaborative interface designed to offer superior control over your interaction with AI models like GPT-3.5 Turbo and beyond. Our app not only allows multiple users or devices to work on a document in real time, but also provides unique control over your prompt history, backend parameters, and offers a series of tools designed to facilitate collaboration and streamline workflow.

Quick Start

Starting a Chat/Prompting Session

To begin a chat session with the AI, you simply need to create a document or join an existing document through a shared link. Once you're on the chat interface page, you can start entering your prompts for the AI to respond.

Chat Session User Interface

Overview

The chat session interface forms the heart of the PromptPlus AI experience. It's where you interact directly with the AI, entering prompts, and receiving responses. This interface provides the primary controls for your interaction with the AI.

Component Description
Delete This option allows you to delete any previous response.
Selection Use these input boxs to selectively exclude messages sent to the AI. Deselecting prompts allows you to remove or bring back the content into context quickly
Rerun Rerun a prompt with the same or different parameters. Rerun works by moving selected message to the front of the request queue and rerunning it based on your current engine parameters. For best results, select on the message you want in the context history
Copy This option lets you copy any prompt or response to your clipboard.
Token visualizer A visualizer at the top of the text area helps understand the model's tokenization process, providing insight into how text is broken down. Chatgpt uses tokens in its model and uses tokens to track cost. Cost per token are on a per model basis

ChatGPT models have a context limit that you want to be aware of. This limit is not readily apparent on the ChatGPT interface so many users are not aware however it may lead to the AI not responding predictably.

The chat interface features are designed to give you as much control over your session as possible. Selecting the most useful responses in a session is imperative for best results. Select, delete, and rerun are the tools to help you do this.

Document Details and Engine

The chat session sidebar, or menu on mobile, houses important document options, engine parameters, and user details.

Document stats

This section provides an document options as well as an overview of the document: title, token count, and time of last activity.

Inside the document options, there are export options, options to edit title/system messages/or upload previous prompts. It display more indepth token usage stats as well as an owner tab for token management and organization purposes

Engine

Here you can choose the chat model and adjust parameters like max response tokens, temperature, top_P, presence penalty, and frequency penalty.

Currently offering the following Chat Models:

MODEL MAX TOKENS DESCRIPTION
gpt-3.5-turbo 4,096 tokens

Can do any language task with better quality, longer output, and consistent instruction-following than the curie, babbage, or ada models.
gpt-3.5-turbo-16k 16,384 tokens

Same capabilities as the standard gpt-3.5-turbo model but with 4 times the context.

Chatgpt Parameters

Changes from default values are indicated with color cues.
Yellow denotes changes in max tokens, while orange represents modifications in other parameters.

PARAMETER DESCRIPTION VALUE NOTES
max_token The maximum number of tokens that the AI model is allowed to generate in response to a particular prompt model dependent affects the cost per API call and the time it takes to get a response

Very long conversations may receive incomplete replies because each model version (like GPT-3.5 Turbo) has a maximum token limit it can handle per conversation. For example, GPT-3.5 Turbo has a limit of 4096 tokens per conversation, while other models may have different limits. Understanding and adjusting the Max Tokens parameter is essential for managing the trade-off between the depth of the AI's responses and the resources (time and cost) required for each interaction.

PARAMETER DESCRIPTION VALUE NOTES
temperature Governs the randomness of responses. Higher values create diverse but potentially less focused replies, while lower values yield more focused and predictable answers. 0 to 2 Can be thought of as creativity dial. Generally recommended altering this or top_p but not both.
top_p Controls the word range for the AI's response. Lower values make it stick to more common words, whereas higher values allow it to consider a broader vocabulary. 0 to 1 Generally recommended altering this or top_p but not both.
Frequency Penalty High values discourage the AI from repeating itself, whereas low values make the AI more likely to reiterate previous phrases or sentences -2.0 to 2.0 only apply when there is existing conversation history. They don't affect the AI's initial responses in a new conversation
Presence Penalty High values encourage the AI to bring up new topics in the conversation, while lower values make the AI stick to the existing topics. -2.0 to 2.0 only apply when there is existing conversation history. They don't affect the AI's initial responses in a new conversation

Note Promptplus AI simplies values into a slider bar. Values are for informative purposes only. For more information on these parameters, visit OpenAI's documentation.

Dashboard/Sessions Application Screen

Overview

The Sessions page is your hub to manage and access all documents/sessions created or shared with you. This section provides a comprehensive overview of all the sessions in a user-friendly file system.

Here are the key components of each line item, from left to right:

Share Icon

Click on this icon to get a shareable link for the session.

Icon color indicates whether the session is shared or not.
The color coding is as follows:

  • link indicates you are the owner.
  • link indicates that you are the owner and you have shared the document.
  • link indicates that the session is shared with you (you are not the owner).

Document Title

The title of the document, which you can set during creation or edit later.
If you don't provide a title during creation, the first prompt is used as the document name.

Prompt Counter

Shows the total number of prompts executed in the session.
This feature helps you identify the desired session quickly.

Last Activity Date

Shows the date and time of the most recent prompt request in the session

Labels

Labels can be assigned during creation or added later to help you filter and categorize your sessions. The default labels provided are "business", "personal", and "archived".

Default labels can be changed in your profile settings.

Users

Displays the users in your session and their current online status.

Document Options

This option is also available inside the chat interface at the top by document title. Aside from the prompt tab, most of document options are availble to users initally when they create the session.
The Document Options includes several features:

Prompts Tab

This section contains different export and format options Text, HTML, JSON, CSV
Export the entire chat history or choose specific parts to download.

Options Tab

  • Edit title: change session title
  • Edit system message: A system message gives context to a session without invoking a response.
  • upload: upload previous prompts for a session
  • Delete Session: Permanently delete the session
  • Archive Session: Permanently delete the session
  • Clone Session: Create a duplicate of the current session.
  • Clone Packet: Click to get a JSON of the packet info sent to API
  • Token Usage Stats: Provides detailed statistics on token usage.

Owner Tab

Add more labels. Labels are used to filter by

Add Owners note. Owner notes are only visibile to the owner and for referential purposes only
