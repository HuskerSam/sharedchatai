Introduction link

Unacog is a online AI chat tool, enabling instant collaboration on any device. The app equips you with a comprehensive set of features to optimize your interaction with advanced AI models.

Designed with developers, educators, and professionals in mind, Unacog makes learning to prompt more collaborative and efficient. Whether you need to get the team up to speed or use generative text AI to solve a problem, we have you covered.

Our platform is model-agnostic, supporting both OpenAI and Google LLM models. We are continually expanding our support to include new models as they become available. Model updates will be communicated our homepage under the News tab.

No Password Sign In

Unacog AI employs a passwordless token authentication system for your enhanced security, privacy, and convenience. With no sign-up required, simply leverage credentials you already use across various platforms to log in.

To view a document on Unacog, users must first sign in.

Available login options include:

Email Login:

With this option, you receive a unique login link in your email inbox that provides access to your account. No password is required, all your data is saved to your email which is required for sign in. You may change your email address in the profile menu.

Google Login:

Use your existing Google account to log in. This is our recommended method as it quickly gets you started with your profile image and name from Google.

Anonymous Login:

This is recommended for first-time users whose only intention is to view documents. Please note that with anonymous login, you won't be able to revisit your profile, as this option doesn't create a permanent account.

Using the Chat Interface link

The chat session interface where you interact directly with the AI, entering prompts, receiving responses, and leveraging powerful features designed to optimize the AI's performance.

Designed for Collaboration

In a shared session, Unacog AI enables all participants to interact with the AI model via a unified chat interface. Usernames distinctly label each contribution, maintaining clarity throughout the dialogue. Features available to all participants include:

Prompting

Prompting is the process of entering text into the AI model to generate a response. The AI's response is based on the context of the prompt and the parameters set for the engine.

The color coding of the session provides instant visual cues, enabling users to identify changes in parameters swiftly and thus ensuring effective and informed interactions with the AI.

YELLOW Indicates a modification in the 'Max_token' parameter.

ORANGE Reflects adjustments to any parameter that could influence your outcome. These parameters vary based on the model in use. For OpenAI's chatgpt, this includes changes to temperature, top_p, frequency penalty, or presence penalty.

RED Serves as a warning that the app anticipates your next message may reach the API's token cap limit.

Prompt Inclusion/Exclusion

Each user can determine which prompts to include or exclude from the AI's response context, controlled by a selection box attached to each prompt.

Exclusion of prompts allows users to modify the context swiftly. Each participant can influence the AI's consideration when generating responses, granting collective control over the conversation and narrative. Auto exclude can be enabled in profile options.

Prompt Rerun loop

Users have the option to rerun any included prompts to obtain a fresh AI response. Rerun a prompt with the same or different parameters, particularly useful if the initial AI response was suboptimal or if alternate perspectives on the same prompt are desired. Rerun operates by placing the selected message at the front of the request queue and resubmitting it based on the current engine parameters. For optimum outcomes, include the desired message in the context history. When rerunning another user's message, the ownership of the message shifts to you.

Change Engine Settings

All users are equipped to modify the AI's response parameters, like temperature and top_p, customizing the AI responses to be more creative, more factual, or anywhere in between. Please refer to the parameters section below for details.

Delete Prompts

Delete removes a prompt from the session chat. While anyone may deselect or rerun a prompt, only message owners are able to delete. This feature is useful for removing prompts that are no longer relevant or that you no longer wish to be included in the AI's context.

Feedback on Token Usage

In addition to providing a platform for interaction with AI, the chat interface is equipped with stats and features that provide feedback on token usage, aiding in cost management and comprehension of the model's processing.

Token Visualizer

Positioned at the top of the text area, the token visualizer offers a graphical representation of the model's tokenization process, providing insights into how text is dissected into tokens. Tokens are integral to how GPT models operate and calculate cost - the cost per token varies on a per-model basis.

Related Token limits

Total Tokens

The total token count for each prompt, encompassing both the user's input and AI's response, is displayed at the top of each message, below the activity timer. This aids in tracking the total token usage and managing the context that the AI can keep for subsequent responses.

Completion/Reponse Tokens

For each prompt or message, the number of completion tokens utilized is displayed at the top right of each message. Completion tokens are the number of tokens used in the AI's response to a particular prompt. This helps you understand how much of the model's capacity was used for generating each response.

Activity Timer

Each message is accompanied by an activity timer that shows the time elapsed since the message was sent. This timer updates every time the page is loaded, giving a clear indication of the age of each message. For messages older than 24 hours, the time is delineated in a month (3 character abbreviation) and day format.

Chatgpt Parameters

Changes from default values are indicated with color cues.

  • max_token
    The maximum number of tokens that the AI model is allowed to generate in response to a particular prompt. May affect the cost per API call and the time it takes to get a response.
    model dependent

Related Token limits

  • temperature

    Governs the randomness of responses. Higher values create diverse but potentially less focused replies, while lower values yield more focused and predictable answers.

    0 to 2
  • Controls model's response predictability
  • Lower temperature = more common responses
  • Higher temperature = less predictable, more creative responses
  • Common parameter in most LLMs

If we revisit the "The apple is..." example, with a lower temperature, the model is more likely to predict a common descriptor like "red". However, when you increase the temperature, it amplifies the chance of less common descriptors being chosen, such as "ripe" or "delicious".

  • top_p

    Controls the word range for the AI's response. Lower values make it stick to more common words, whereas higher values allow it to consider a broader vocabulary.

    0 to 1
  • Works with nucleus sampling to control response predictability
  • Lower top_p = exact, fact-based responses
  • Higher top_p = more diverse responses
  • Common parameter in most LLMs

Note Generally recommended altering this or top_p but not both.

  • Frequency Penalty

    This setting discourages the model from using commonly occurring words and phrases in its response, regardless of the specific context.

    -2.0 to 2.0
  • Discourages common words/phrases
  • High frequency_penalty = creative/unique, but possibly less straightforward responses

The frequency penalty discourages the model from using words or phrases that it understands to be common or frequently occurring. This understanding is based on the vast amount of text data from the internet that the model was trained on, not just the immediate context of the conversation.

By increasing the frequency_penalty, you're asking the model to generate responses that avoid these common words and phrases, leading to outputs that might be more creative or unique. However, a high frequency penalty can also make the model's responses less straightforward or harder to understand.

  • Presence Penalty

    This setting influences how much the model should stick to or diverge from the words and phrases present in the current input or context.

    0 to 1
  • Relates to current input/context
  • High presence_penalty = encourages use of input-specific words/phrases, helps model stay on topic
  • Low presence_penalty = introduces new words/ideas/topics, resulting in more varied responses, but possibly less related to initial input

In contrast to frequency_penalty, the presence_penalty directly relates to the current input or context. When set high, it encourages the model to use or stick to the words and phrases that have been mentioned in the conversation or input, helping the model to stay on topic.

Note Don't worry about remembering the values on Unacog AI, we have simply these values with a slider bar. For more information on these parameters, visit OpenAI's documentation.

Getting to Know Large Language Models (LLMs)

Ever wondered how a Large Language Model, like GPT-3, comes up with its responses? Well, let's pull back the curtain a bit.

Think of LLMs as super-smart word predictors. They use the words you've given them to guess what word should come next. For instance, if you say, "The apple is...", it might guess "red" because that's a typical subsequent descriptor in sentences it's encountered during training.

But here's the thrilling part - it's not always "red". These models apply something called a probabilistic approach, which implies the same phrase could yield different responses. So, sometimes, the apple could be "ripe" or even "delicious" - it all hinges on the context!

Understanding how these models work helps us get the best out of them, whether we're teaching a class, exploring a research question, or just having fun. And for those of you who love diving into the nitty-gritty, understanding this foundational concept is key when it comes to fine-tuning how the AI responds - do you want it to be more creative or less repetitive? You'll get the hang of it!

Understanding and Managing Token Limits

Each Large Language Models (LLMs) has a token limit. GPT-3.5 Turbo's limit is 4096 tokens per conversation. This limit dictates the AI's context, meaning once reached, the earliest parts of the conversation are omitted from the AI's consideration.

In many AI interfaces, this isn't made explicit, potentially leading to confusion if the AI responses lack context from earlier messages. Unacog AI addresses this by keeping users informed about their token usage in a session, ensuring they are aware of the AI's current context.

Unacog will alert you to exclude previous messages when we detect you may go over the model's token API limit

Understanding and managing token limits can significantly enhance your AI interaction. You should become aware of how many tokens are used in each message, and what the limit is for the model you are using.

One strategy for managing content is to select which prompts are included or excluded from the AI's context. This gives you control over the information the AI considers, optimizing your interaction within the token limit, and improving response quality.

Understanding token limits will also help you gauge how much it will cost to run something. Usage can add up quickly on more complex models: chat-gpt-3.5-16k is about 2x the cost of gpt-3.5-turbo. While gpt-4 is about 10x more expensive than gpt-3.5-turbo

ChatGPT Prompt Structure

ChatGPT is a versatile AI tool that facilitates interactive dialogues. Its structure can initially seem complex, so we're simplifying it for you.

Here's how it works:

  1. Prompts: Instead of one long string, your input breaks down into a series of messages.
  2. Message List: This list includes all the messages, staying within the token limit.
  3. Attributes: Every message has 'role' and 'content'.
  4. Roles: These can be 'system', 'user', or 'assistant'.
  5. Content: This is the actual text of the message.
  6. System Message: This can set broad instructions for the conversation.
  7. Sequence: The assistant responds to messages in the order they appear.

For instance, the following is an example of how system message can be used to define responses.

'prompt': 'tell me a joke'

To:

'messages': [{'role':'user', 'content':'tell me a joke'}]
System Instructions/Message

System instructions can guide the model's behavior during the conversation.

Consider this example:

'messages': [{'role':'system', 'content':'You are an assistant that speaks like Shakespeare.'}, {'role':'user', 'content':'tell me a joke'}]

This would yield:

{... 'message': {'role':'assistant', 'content':'Why did the chicken cross the road? To get to the other side, but verily, the other side was fraught with danger, so it quickly returned to its safe haven, forsooth!'} ...}
