Formatting Markdown and Code

AI models output data in a large number of formats and mix into their textual response. Text is usually broken down into lines using \n for line break. Markdown and code files are formatted when delimited in the following way:

```
Markdown File
```

the three backquotes denote the start and end of a code block in Markdown format.
Equations are formatted using a KaTeX library. The details of how are KaTeX is delimited varies by model and version, more below.

Marked - for markdown formatting.
KaTeX - for equation formatting.
Highlight.js - for code formatting.


KaTeX block delimiters are enabled by default:
$$
KaTeX
$$

\[
KaTeX
\]

KaTex inline delimiters are disabled by default, as the $ can conflict with currency
\$KaTeX\$

\(KaTeX\)

$KaTeX$

Editing Responses

Often the AI doesn't return quite the right response and you want to adjust it. This can make future responses more on track. In the case of formatted output, responses must be edited in raw format, but will be displayed in html format.


The response will appear in a popup.


The copy button for the response copies the raw response to the clipboard, not the formatted result.
